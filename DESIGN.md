# Linux 二级文件系统设计文档

## 一、概述

本项目是一个基于 EXT2 文件系统设计的 Linux 二级文件系统，实现了完整的文件系统功能，包括文件管理、目录管理、用户管理、回收站、日志系统等。

## 三、数据结构设计

### 3.1 Super Block（超级块）

```c
struct super_block {
    uint32_t s_inodes_count;       /* 总inode数 */
    uint32_t s_blocks_count;       /* 总块数 */
    uint32_t s_free_blocks_count;  /* 空闲块数 */
    uint32_t s_free_inodes_count;  /* 空闲inode数 */
    uint32_t s_magic;              /* 魔数 */
    uint32_t s_block_size;         /* 块大小 */
    uint32_t s_user_count;         /* 用户数 */
};
```

### 3.2 Inode（索引节点）

```c
struct inode {
    uint16_t i_mode;               /* 文件类型和权限 */
    uint16_t i_uid;                /* 所有者UID */
    uint32_t i_size;               /* 文件大小 */
    uint32_t i_ctime;              /* 创建时间 */
    uint32_t i_mtime;              /* 修改时间 */
    uint32_t i_blocks;             /* 占用块数 */
    uint16_t i_links_count;        /* 链接数 */
    uint32_t i_block[15];          /* 块指针 */
};
```

### 3.3 Directory Entry（目录项）

```c
struct dir_entry {
    uint32_t inode;                /* inode号 */
    uint16_t rec_len;              /* 记录长度 */
    uint8_t name_len;              /* 文件名长度 */
    uint8_t file_type;             /* 文件类型 */
    /* 文件名紧跟其后 */
};
```

### 3.4 用户表

```c
struct user_entry {
    char name[MAX_USERNAME];       /* 用户名 */
    char passwd[MAX_USERNAME];     /* 密码 */
    uint32_t uid;                  /* 用户ID */
};
```

## 四、磁盘布局

```
块0: 引导块
块1: 超级块 + 组描述符
块2-13: 块位图 (12块)
块14-141: inode位图 + inode表 (128块)
块142-143: 用户表 (2块)
块144-159: 日志区域 (16块)
块160-32767: 数据区
```

## 五、核心算法

### 5.1 块分配算法

采用顺序扫描算法，从数据区起始块开始扫描块位图，找到第一个空闲块。

### 5.2 Inode分配算法

采用顺序扫描算法，从 inode 1 开始扫描 inode 位图，找到第一个空闲 inode。

### 5.3 目录查找算法

通过遍历目录项，匹配文件名找到对应的 inode 号。

### 5.4 块缓存算法

采用 FIFO 替换策略的块缓存，大小为 32 块。

## 六、功能特性

### 6.1 文件操作

- 创建文件：`create`
- 删除文件：`delete`
- 修改权限：`chmod`
- 打开文件：`open`
- 关闭文件：`close`
- 读取文件：`read`
- 写入文件：`write`

### 6.2 目录操作

- 创建目录：`mkdir`
- 删除目录：`rmdir`
- 切换目录：`cd`
- 列出目录：`dir`

### 6.3 链接操作

- 创建硬链接：`ln`
- 创建软链接：`ln -s`

### 6.4 用户管理

- 用户登录：`login`
- 用户登出：`logout`
- 用户注册：`register`

### 6.5 回收站

- 移入回收站：`trash-move`
- 查看回收站：`trash`
- 恢复文件：`trash-restore`
- 清空回收站：`trash-empty`

### 6.6 日志系统

- 查看日志：`journal`
- 清空日志：`journal-clear`
- 切换日志：`journal-toggle`

## 七、错误处理

### 7.1 边界检查

- inode 号范围检查
- 块号范围检查
- 文件名长度检查
- 路径长度检查

### 7.2 错误返回

- 返回 -1 表示失败
- 返回 0 表示成功
- 错误信息输出到 stderr

## 八、性能优化

### 8.1 块缓存

采用 FIFO 替换策略，减少磁盘 I/O 次数。

### 8.2 预分配

文件创建时预分配数据块，减少后续分配开销。

### 8.3 延迟写入

脏块延迟写入磁盘，提高写入性能。

## 九、安全特性

### 9.1 用户认证

用户名/密码认证机制。

### 9.2 权限控制

基于 inode 权限位的访问控制。

### 9.3 日志记录

所有操作记录到日志，便于审计和恢复。
